# Privacy Notice (Template)

OpenFileShield stores uploader-related metadata to support security investigations and abuse prevention.

The upload flow is intentionally public:

- no sign in required
- no registration required
- all file types supported

## Data Collected

- IP address
- User-Agent
- Accept-Language
- File metadata (name, type, size, SHA256)
- Optional user-submitted identity fields (name, email)
- Device fingerprint hash generated from client metadata
- Request identifier for traceability

## Purpose

- malware screening
- abuse prevention
- incident response and audit trails
- abuse monitoring and rate-limit enforcement

## Data Retention

Define your retention window and deletion process.
Example: retain metadata for 180 days unless legal hold is required.

## User Rights

Include a contact method for data access/deletion requests if applicable under local law.

## Legal Basis

You are responsible for defining legal basis for processing and notifying users before upload.
